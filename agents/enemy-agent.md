# Enemy Agent

## 목적
- 일반몹/엘리트/보스 설계와 웨이브 운영의 기준 문서다.
- 목표는 "공정한 압박 + 명확한 대응 + 점진적 난이도 상승"이다.

## 적의 역할 정의
- 적은 플레이어를 즉사시키는 장치가 아니라 "위치 선택 비용"을 만드는 시스템 요소다.
- 위험 신호(텔레그래프)와 대응 창구(회피/차단/거리 확보)를 반드시 제공한다.

## 분류 기준
- 일반몹: 기본 압박, 수량으로 긴장 형성
- 엘리트: 패턴 변주, 조합 시너지, 전투 리듬 전환
- 보스: 페이즈 기반 목표전, 메커니즘 학습 시험

## 설계 원칙
- 읽히는 패턴: 처음 보는 패턴도 2~3회 내 학습 가능해야 한다.
- 회피 가능성: 피격은 플레이어 실수 결과여야 한다.
- 조합 안정성: 몹 조합이 회피 불가능 상태를 만들면 안 된다.
- 보상 정합성: 위험이 높을수록 보상(경험치/드랍/공간 이득)이 따라야 한다.

## 웨이브/난이도 운영
- 시간 구간별 목표 난이도 명시
- 스폰 상한선(동시 개체 수) 고정
- 난이도 증가는 HP/공격력만이 아니라 패턴/배치로 조절

## 보스 설계 체크
- [ ] 페이즈 전환 조건 명시 (체력 %, 시간, 기믹)
- [ ] 각 페이즈 텔레그래프 존재
- [ ] 무적/회피 불가 구간 최소화
- [ ] 보상 및 클리어 피드백 명확

## 구현 체크리스트
- [ ] 적 타입/스탯/행동 상태 정의
- [ ] 스폰 규칙(최소/최대, 쿨다운) 정의
- [ ] 충돌/피격/사망 이벤트 연결
- [ ] 드랍/보상 테이블 연결
- [ ] 문서(웨이브표/패턴표) 동기화

## 성능 체크
- 경로탐색/타겟팅 연산 빈도 제한
- 동시 개체 수 상한 유지
- VFX/SFX 폭주 방지(중첩 상한)

## 리뷰 우선순위
- `critical`: 회피 불가 즉사 패턴, 진행 차단
- `high`: 난이도 급변, 패턴 판독 불가
- `medium`: 보상 불균형, 일부 조합 과도
- `low`: 텔레그래프 타이밍/연출 개선

## 완료 기준
- 사망 원인을 플레이어가 설명할 수 있다.
- 웨이브 곡선이 급격히 깨지는 구간이 없다.
- 보스 페이즈별 대응 전략이 존재한다.
