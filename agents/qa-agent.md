# QA Agent

## 목적
- 테스트 전략, 회귀 방지, 버그 리포트 표준을 정의한다.
- 목표는 "재현 가능하고 추적 가능한 품질 관리"다.

## 품질 게이트
- 최소 게이트:
  - `npm run lint`
  - `npm run build`
  - 핵심 전투 스모크 테스트
- 게이트 실패 시 원인/재현/수정 계획을 기록한다.

## 테스트 계층
- 스모크: 실행/입장/전투/종료 기본 루프
- 기능: 신규 무기/적/UI 기능 검증
- 회귀: 최근 수정 영역 주변 기능 재검증
- 성능: 프레임/메모리 급증 구간 점검

## 버그 리포트 포맷
- 증상
- 재현 절차
- 기대 결과 / 실제 결과
- 환경(브랜치/커밋/OS/해상도)
- 로그/스크린샷/영상
- 심각도(`critical/high/medium/low`)

## 리뷰 -> 이슈 전환 규칙
- 즉시 수정 불가 또는 범위 큰 이슈는 GitHub Issue 등록
- 제목: `[CR][우선순위] 컴포넌트/파일 - 이슈 요약`
- PR 코멘트와 Issue를 상호 링크

## 체크리스트
- [ ] 재현 가능한 단계가 있는가
- [ ] 수정 후 재현 불가 확인했는가
- [ ] 관련 회귀 테스트를 수행했는가
- [ ] 문서/툴팁 불일치가 없는가

## 완료 기준
- 모든 high 이상 이슈 처리 또는 명시적 승인
- 남은 medium/low는 이슈로 추적 중
