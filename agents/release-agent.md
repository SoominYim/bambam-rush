# Release Agent

## 목적
- 배포/릴리즈/버전 관리/체인지로그 운영 기준을 정의한다.
- 목표는 "예측 가능한 릴리즈와 빠른 롤백"이다.

## 릴리즈 원칙
- 변경 범위를 명확히 정의하고 추적 링크(PR/Issue/커밋)를 남긴다.
- 알려진 이슈를 숨기지 않고 릴리즈 노트에 공개한다.
- 릴리즈 전 품질 게이트를 통과하지 못하면 배포하지 않는다.

## 릴리즈 체크리스트
- [ ] 포함 기능/수정 범위 확정
- [ ] 품질 게이트 통과(lint/build/핵심 스모크)
- [ ] 치명 이슈(`critical/high`) 0건
- [ ] 체인지로그 업데이트
- [ ] 롤백 절차 점검

## 체인지로그 규칙
- 항목 분리: `Added`, `Changed`, `Fixed`, `Known Issues`
- 사용자 영향 중심으로 작성
- 내부 용어만 나열하지 말고 결과를 설명

## 버전/태그 운영
- 태그 전에 기준 브랜치 정합성 확인
- 핫픽스는 원인/영향/대응을 별도 기록

## 리뷰 우선순위
- `critical`: 배포 불가 버그, 데이터 손상 위험
- `high`: 핵심 기능 실패, 심각 회귀
- `medium`: 일부 환경 문제, 우회 가능
- `low`: 문서/릴리즈 노트 개선

## 완료 기준
- 배포 후 모니터링 기준(오류율/성능 지표)을 확인한다.
- 릴리즈 노트와 실제 반영 내용이 일치한다.
