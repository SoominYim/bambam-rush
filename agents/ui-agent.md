# UI Agent

## 목적
- HUD/메뉴/튜토리얼/UI 피드백의 일관된 기준을 정의한다.
- 목표는 "전투 중 1초 내 핵심 정보 인지"다.

## 정보 우선순위
1. 생존 정보: HP, 피해 방향, 위험 경고
2. 전투 정보: 주무기 상태, 쿨다운, 군중 상태
3. 성장 정보: 레벨업 선택, 빌드 시너지 힌트
4. 보조 정보: 로그/기록/설정

## 설계 원칙
- 가독성 우선: 장식보다 정보 전달 우선
- 일관성: 동일 액션은 동일 위치/스타일/상호작용
- 접근성: 색상만으로 상태를 구분하지 않는다
- 반응성: 입력 -> 시각 반응 지연 최소화

## 전투 UI 체크
- [ ] 피격 시 즉시 피드백(숫자/플래시/사운드)
- [ ] 위험 영역/보스 패턴 경고 가시성 확보
- [ ] 쿨다운/충전 상태 직관적 표시
- [ ] 상태 이상 아이콘과 지속시간 표시

## 메뉴/흐름 체크
- [ ] 시작 -> 전투 -> 결과 -> 재시작 흐름 단절 없음
- [ ] 키보드/마우스 모두 조작 가능
- [ ] 튜토리얼 문구와 실제 조작 일치

## 해상도/플랫폼 체크
- [ ] 16:9, 16:10, 저해상도에서 레이아웃 붕괴 없음
- [ ] 최소 클릭/터치 영역 확보
- [ ] DPI/스케일 변경 시 겹침 없음

## 리뷰 우선순위
- `critical`: 핵심 정보 미표시, 진행 불가 UI
- `high`: 전투 피드백 누락, 오해 유발 인터랙션
- `medium`: 가독성 저하, 배치 비일관성
- `low`: 여백/정렬/문구 개선

## 완료 기준
- 신규 유저가 튜토리얼 없이도 기본 전투 루프를 이해한다.
- 전투 중 사망 원인을 UI로 추적 가능하다.
